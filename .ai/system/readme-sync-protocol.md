# README 同步协议

## 目标

确保 `.ai/` 目录下所有 `README.md` 文件始终与实际目录结构、工程进度保持一致。每次迭代完成后，Agent 必须检查并更新受影响的 README 文件。

---

## 适用范围

本协议在以下场景强制执行：

1. **新增/删除/重命名文件**：涉及 `.ai/` 目录下任何文件变更
2. **工程进度变更**：模块状态从"待开发"变为"已完成"等
3. **目录结构变更**：新增子目录、移动文件等
4. **提示词体系变更**：重构、重写、归档提示词文件

---

## 需要同步的 README 文件

| 文件 | 职责 | 更新触发条件 |
|------|------|------------|
| `.ai/README.md` | 提示词体系总览，包含完整目录树 | 任何文件/目录变更 |
| `.ai/prompts/08-platform/README.md` | 租户平台业务总览，包含模块状态 | 后端/前端模块状态变更、新增模块 |
| `.ai/prompts/11-base-library/README.md` | 底层框架工程说明 | 底层工程调整任务完成 |
| `.ai/prompts/12-business-template/README.md` | 业务模块模板说明 | 通用层引用变更 |

---

## 检查清单（强制逐项确认）

每次迭代结束时，Agent 必须对照以下清单执行 README 同步：

### 1. `.ai/README.md` 目录树

- [ ] 目录树中的文件列表与实际 `find .ai/ -type f | sort` 输出一致
- [ ] 新增文件已添加到目录树中
- [ ] 删除/重命名的文件已在目录树中更新
- [ ] `tasks/` 目录下的任务文件已列出
- [ ] `workspace/` 目录描述准确

### 2. `.ai/prompts/08-platform/README.md` 模块状态

- [ ] 每个模块的"后端状态"列与实际后端代码状态一致
- [ ] 每个模块的"前端提示词状态"列与实际提示词重写进度一致
- [ ] 基础设施表中的状态正确
- [ ] 技术栈部分的项目路径正确
- [ ] 顶层模块概要文件表与实际存在的 `.md` 文件一致

### 3. 交叉引用完整性

- [ ] 所有 README 中引用的文件路径实际存在
- [ ] 不存在引用已删除/已重命名文件的死链接

---

## 执行时机

| 时机 | 动作 |
|------|------|
| 每次迭代结束 | 对照检查清单更新所有受影响的 README |
| 写会话总结前 | 确认 README 已同步，在总结中记录同步情况 |
| 文件重命名/批量操作后 | 立即更新 README 目录树 |

---

## 与自审协议的关系

本协议是 `.ai/system/self-review-protocol.md` 的补充。自审协议负责代码层面的审查，本协议负责文档层面的一致性。两者都是迭代收尾阶段的强制步骤。

---

## 禁止事项

1. **禁止** README 中使用通配符（如 `00xx_*-page.md`）代替完整文件列表
2. **禁止** 保留已删除文件的引用
3. **禁止** 使用过时的项目路径或模块状态
4. **禁止** 跳过 README 同步直接结束迭代
