# 审查策略

## 目标

定义 Agent 产出的审查流程和标准，确保代码质量、安全性和一致性。

---

## 适用范围

所有 Agent 产出的代码、测试、文档。

---

## Agent 自检清单

Agent 在标记任务完成之前，必须逐项自检：

### 后端自检项

- [ ] 编译通过：`dotnet build YTStd.slnx` 无错误
- [ ] 测试通过：`dotnet test YTStd.slnx` 全部通过
- [ ] 无反射、无 `dynamic`、无 LINQ
- [ ] 所有 `InsertAsync` 前有 `DB.GetNextLongIdAsync()`
- [ ] 所有 `Logger.Debug` 使用 `Func<string>` 重载
- [ ] 所有错误消息使用 `Messages.XXX` 常量
- [ ] 无裸 `TenantId` / `tenant_id` 字段名
- [ ] 所有公开类型/方法有中文 XML 注释
- [ ] JSON 使用 `Utf8JsonWriter` 或 `JsonSerializerContext`
- [ ] 中间件错误响应使用 PascalCase JSON 属性名
- [ ] 唯一索引实体有 check-exists 端点
- [ ] `ApiResult` 使用 `Code`/`Message`/`Data` PascalCase

### 前端自检项

- [ ] 编译通过：`npm run build` 无错误
- [ ] 类型使用 PascalCase 匹配后端
- [ ] API 封装使用 `ApiResult<T>` 格式
- [ ] 所有页面有功能说明和操作指引
- [ ] 权限控制正确
- [ ] 路由注册且有守卫
- [ ] DevExtreme 组件使用正确

### 通用自检项

- [ ] 无硬编码中文字符串作为错误消息
- [ ] 无 console.log / print 残留（前端）
- [ ] 无 TODO 残留（除非有明确标记和说明）
- [ ] 文件组织符合目录规范
- [ ] 命名符合命名规范

---

## 人类审查重点

人类开发者在审查 Agent 产出时，重点关注：

### 高优先级

1. **安全性**：认证授权逻辑是否正确
2. **数据一致性**：事务边界是否正确
3. **业务逻辑**：是否符合业务需求
4. **性能**：是否有明显的性能问题

### 中优先级

5. **测试覆盖**：关键路径是否有测试
6. **错误处理**：异常场景是否处理
7. **国际化**：i18n 是否正确接入

### 低优先级

8. **代码风格**：命名和格式是否统一
9. **注释质量**：注释是否准确
10. **文档完整**：API 文档是否完整

## 审查反馈格式

人类审查后，使用以下格式记录反馈：

```markdown
## 审查反馈

### 文件：{文件路径}

| 行号 | 级别 | 问题描述 | 建议修改 |
|-----|------|---------|---------|
| 42 | 严重 | 缺少权限检查 | 添加权限中间件 |
| 87 | 建议 | 变量命名不清晰 | 重命名为 xxx |

### 整体评价
- 通过 / 需修改 / 需重写

### 修改指令
- {给 Agent 的具体修改指令}
```

---

## 审查触发条件

以下场景必须触发人类审查：

1. 新增或修改实体结构
2. 新增或修改认证/权限逻辑
3. 新增数据库表或索引
4. 修改中间件管道
5. 新增外部依赖
6. 修改全局配置
7. 涉及数据删除的操作
8. 涉及金额/计费的逻辑
