# 国际化规范

## 目标

统一前端和后端的国际化职责分工、实现方式和资源管理规范。

---

## 适用范围

所有涉及多语言的前端和后端代码。

---

## 支持的语言

| 语言 | 代码 | 后端枚举 | 前端 locale | 备注 |
|------|------|---------|------------|------|
| 简体中文 | zh-CN | `Lang.ZhCn (0)` | `zh-CN` | 基准语言 |
| 英语 | en-US | `Lang.En (1)` | `en-US` | 国际化首选 |
| 日语 | ja-JP | `Lang.Ja (2)` | `ja-JP` | — |
| 繁体中文 | zh-TW | `Lang.ZhTw (3)` | `zh-TW` | — |
| 马来语 | ms-MY | `Lang.MsMy (4)` | `ms-MY` | — |

默认语言：简体中文（zh-CN）。

## 前后端职责分工

### 后端职责

| 内容 | 后端负责 | 说明 |
|------|:-------:|------|
| API 错误消息 | ✓ | 使用 `Messages.XXX` 常量（i18n key） |
| 业务规则错误 | ✓ | 使用 `Messages.XXX` 常量 |
| 日志消息 | ✗ | 日志内容不需要国际化 |
| 数据库字段值 | ✗ | 数据库存储原始值 |
| 枚举显示名 | ✗ | 前端负责枚举的显示翻译 |

### 前端职责

| 内容 | 前端负责 | 说明 |
|------|:-------:|------|
| 菜单文本 | ✓ | `$t('menu.xxx')` |
| 按钮文本 | ✓ | `$t('common.xxx')` |
| 表单标签 | ✓ | `$t('views.xxx.form.xxx')` |
| 表格列标题 | ✓ | `$t('views.xxx.columns.xxx')` |
| 验证消息 | ✓ | `$t('validation.xxx')` |
| 枚举显示名 | ✓ | `$t('enums.xxx')` |
| 功能说明 | ✓ | `$t('views.xxx.description.xxx')` |
| 操作指引 | ✓ | `$t('views.xxx.guide.xxx')` |
| 日期时间格式 | ✓ | 使用 `Intl.DateTimeFormat` |
| 数字格式 | ✓ | 使用 `Intl.NumberFormat` |
| 货币格式 | ✓ | 使用 `Intl.NumberFormat` |
| 后端错误消息翻译 | ✓ | 后端返回 i18n key，前端翻译显示 |

## 后端国际化实现

### Messages 常量

```csharp
// Application/Constants/Messages.cs
public static class Messages
{
    // 通用
    public const string Success = "common.success";
    public const string NotFound = "common.not_found";
    public const string OperationFailed = "common.operation_failed";

    // 认证
    public const string InvalidCredentials = "auth.invalid_credentials";
    public const string TokenExpired = "auth.token_expired";

    // 用户
    public const string UsernameExists = "user.username_exists";
    public const string UserNotFound = "user.not_found";
    public const string UserDisabled = "user.disabled";
}
```

### 使用方式

```csharp
// 正确 — 使用 Messages 常量
return ApiResult.Fail(ErrorCodes.UsernameExists, Messages.UsernameExists);

// 错误 — 禁止硬编码中文
return ApiResult.Fail(ErrorCodes.UsernameExists, "用户名已存在");

// 错误 — 禁止直接使用 i18n key 字符串
return ApiResult.Fail(ErrorCodes.UsernameExists, "user.username_exists");
```

### YTStdI18n 使用

```csharp
// 服务端需要翻译时（如邮件模板、通知内容）
var text = I18n.T(tenantId, LangKeys.SomeKey);
```

### 后端错误消息翻译

后端返回 i18n key，前端负责翻译后显示：

```typescript
// utils/http.ts
function handleError(result: ApiResult) {
  if (result.Code !== 0) {
    const message = t(result.Message) || result.Message;
    showNotification({ message, type: 'error' });
  }
}
```

## 回退策略

### 前端回退

1. 查找当前 locale 的翻译
2. 如果未找到，回退到 `zh-CN`（基准语言）
3. 如果仍未找到，显示 key 本身

### 后端回退

1. 查找租户配置的语言翻译
2. 如果未找到，回退到简体中文（`Lang.ZhCn`）
3. 如果仍未找到，返回空字符串

## 不需要国际化的内容

| 内容 | 原因 |
|------|------|
| 代码注释 | 面向开发者，统一使用中文 |
| 日志消息 | 面向运维，不需要多语言 |
| 数据库字段名 | 技术标识 |
| API 路径 | 技术标识 |
| 错误码数字 | 技术标识 |
| 配置键 | 技术标识 |
| 内部常量 | 技术标识 |

## DevExtreme 组件本地化

DevExtreme 有独立的 locale 机制：

```typescript
import { locale, loadMessages } from 'devextreme/localization';
import zhMessages from 'devextreme/localization/messages/zh';

loadMessages(zhMessages);
locale('zh');
```

DevExtreme 的组件内置文本（如分页器、过滤器、排序提示）通过 DevExtreme 自身的 locale 机制处理，不通过 vue-i18n。

## AI 开发时的 i18n 检查

Agent 在开发时必须确保：

1. 所有 `ApiResult.Fail()` 的 message 参数使用 `Messages.XXX`
2. 所有前端用户可见文本使用 `$t()` 或 `t()`
3. 所有新增的 i18n key 在基准语言文件中有对应条目
4. 所有新增的错误消息在 `Messages.cs` 中有对应常量
5. 新增模块时同步更新所有语言的资源文件
