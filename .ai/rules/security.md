# 安全规范

## 目标

定义后端和前端的安全实现标准，防止常见安全漏洞。

---

## 适用范围

所有前端和后端代码。

---

## 认证安全

### Token 管理

- 使用 JWT 或等效的无状态 Token
- Token 必须包含签名并验证
- Token 必须设置有效期
- 刷新 Token 必须有独立的过期时间
- Token 中不存储敏感信息（密码、完整权限树等）

### 密码安全

- 密码必须使用安全哈希算法存储（如 bcrypt、Argon2）
- 禁止明文存储密码
- 禁止可逆加密存储密码
- 密码策略：最小长度 8 位、包含大小写字母和数字
- 支持登录失败锁定机制

### 登录安全

- 限制登录失败次数
- 失败次数超限后锁定账户或增加等待时间
- 记录登录日志（IP、时间、结果）
- 支持 MFA（可选）

---

## 授权安全

### 权限检查

- 所有非公开端点必须通过权限中间件检查
- 权限检查不可被绕过
- 超级管理员跳过权限检查
- 数据权限不仅检查操作权限，还需检查数据所有权

### 路由保护

- 前端路由必须有权限守卫
- 无权限的路由不显示在菜单中
- 直接访问无权限路由显示 403 页面

---

## 输入安全

### 服务端验证

- 所有客户端输入必须在服务端验证
- 禁止信任客户端发送的权限信息
- 数字类型检查范围
- 字符串类型检查长度
- 枚举类型检查有效值
- ID 类型检查有效性

### SQL 注入防护

- 所有 SQL 必须使用参数化查询（通过 YTStdSqlBuilder）
- 禁止字符串拼接 SQL
- 禁止将用户输入直接嵌入 SQL

### XSS 防护

- 前端渲染用户输入内容时使用文本模式（非 `v-html`）
- 必须使用 `v-html` 时对内容进行消毒处理
- HTTP 响应设置 `Content-Security-Policy` Header

---

## 数据安全

### 敏感数据

- 禁止在代码中硬编码密码、密钥、连接字符串
- 敏感配置通过环境变量或安全配置提供
- 日志中禁止输出密码、Token、密钥等敏感信息
- API 响应中不返回密码哈希

### 数据传输

- 生产环境必须使用 HTTPS
- API 之间通信使用 HTTPS
- 敏感数据在传输中加密

---

## API 安全

### 限流

- 所有 API 必须有限流保护
- 登录接口限流更严格
- 批量操作接口限制并发

### CORS

- 生产环境限制允许的 Origin
- 不使用 `*` 通配符

### 错误信息

- 禁止在错误响应中暴露系统内部信息
- 禁止返回数据库错误详情
- 禁止返回堆栈信息（生产环境）
- 使用通用错误消息 + 错误码

---

## 审计

### 必须审计的操作

- 登录/登出
- 用户创建/修改/删除
- 角色授权变更
- 租户状态变更
- 权限配置变更
- 系统配置变更

### 审计记录内容

- 操作人 ID
- 操作时间
- 操作类型
- 操作对象（实体类型 + ID）
- 操作前后数据差异（可选）
- 请求 IP
- TraceId

---

## 安全红线

以下行为绝对禁止：

1. 硬编码密码、密钥、Token
2. 明文存储密码
3. 跳过认证检查
4. 跳过权限检查
5. SQL 拼接
6. 信任客户端权限声明
7. 日志输出敏感信息
8. 生产环境暴露堆栈信息
9. 使用 `v-html` 渲染未消毒的用户输入
10. 禁用 HTTPS（生产环境）
