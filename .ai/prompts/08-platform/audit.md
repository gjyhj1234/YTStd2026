# 审计模块提示词

## 目标

实现操作审计和日志管理能力。

---

## 适用范围

实现审计日志、操作日志、登录日志功能时使用。

---

## 前置阅读

- `.ai/rules/backend.md`
- `.ai/rules/security.md`
- `.ai/rules/database.md`

---

## 输入

- 审计需求
- 已完成的审计中间件

---

## 输出

- `Application/Services/AuditAppService.cs`
- `Endpoints/AuditEndpoints.cs`
- 表：`log_audit`、`log_operation`、`log_login`

---

## 日志类型

| 类型 | 表 | 记录内容 |
|------|---|---------|
| 审计日志 | `log_audit` | 数据变更记录 |
| 操作日志 | `log_operation` | 关键业务操作 |
| 登录日志 | `log_login` | 登录/登出事件 |
| 系统日志 | 文件 | 应用运行日志（由 YTStdLogger 处理） |

---

## 审计记录字段

| 字段 | 说明 |
|------|------|
| 操作人 ID | 执行操作的用户 |
| 操作时间 | 操作发生时间 |
| 操作类型 | 创建/更新/删除/状态变更 |
| 操作对象类型 | 实体类型名 |
| 操作对象 ID | 实体 ID |
| 请求 IP | 客户端 IP |
| TraceId | 请求追踪 ID |
| 变更内容 | 变更前后数据（JSON） |

---

## 约束

- 审计日志只读，不允许修改和删除
- 审计日志支持分页查询
- 审计日志支持按操作人、操作类型、时间范围筛选
- 变更内容使用 JSON 格式存储

---

## 验收标准

- [ ] 审计日志记录完整
- [ ] 日志查询支持分页和筛选
- [ ] 日志数据只读
- [ ] 编译通过
