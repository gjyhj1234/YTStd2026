## 执行步骤

### 第 1 步：创建 `locales/index.ts`（核心 — 组件级文件自动加载）

按照 `05-i18n/frontend-i18n.md` 中的「locales/index.ts 实现模板」创建：

1. 静态导入：主语言文件（5 种）、common（5 种）、generated（5 种）、enums（5 种）
2. 使用 `import.meta.glob` 自动加载所有 `*.vue.{locale}.json`（5 种语言各一组 glob）
3. 实现 `mergeComponentMessages` 函数，过滤 null 值
4. 消息合并顺序（从低到高）：common → generated → enums → 主语言文件 → 组件级
5. 创建 vue-i18n 实例：`legacy: false`、`globalInjection: true`、`locale: 'zh-CN'`、`fallbackLocale: 'en-US'`

**必须实现的函数**：

```typescript
function mergeComponentMessages(modules: Record<string, Record<string, string | null>>): Record<string, string> {
  const result: Record<string, string> = {}
  for (const data of Object.values(modules)) {
    if (!data) continue
    for (const key of Object.keys(data)) {
      const value = data[key]
      if (value !== null && value !== undefined) {
        result[key] = value
      }
    }
  }
  return result
}
```

### 第 2 步：创建 common 资源文件

在 `locales/common/` 下创建 5 个语言文件，包含全局复用文本：

| 类别 | key 示例 |
|------|---------|
| 通用按钮 | 查询、重置、新增、编辑、删除、保存、确定、取消、导出、导入 |
| 通用状态 | 已启用、已禁用、已锁定 |
| 通用操作提示 | 创建成功、更新成功、删除成功、操作成功、操作失败 |
| 通用确认 | 确认删除、确认启用、确认禁用 |
| 通用表格 | 暂无数据、加载中、操作 |
| 通用表单 | 请输入、请选择 |

### 第 3 步：创建 enums 资源文件

在 `locales/enums/` 下创建 5 个语言文件，包含后端枚举值翻译。

### 第 4 步：创建主语言文件

在 `locales/` 下创建 5 个主语言文件（`zh-CN.json` 至 `zh-TW.json`），**仅包含嵌套结构键**：

```json
{
  "app": { "title": "租户管理平台", "logout": "退出", "profile": "个人设置" },
  "languages": { "zh-CN": "简体中文", "en-US": "English", "ja-JP": "日本語", "ms-MY": "Bahasa Melayu", "zh-TW": "繁體中文" },
  "menu": { "dashboard": "仪表盘", "platformManagement": "平台管理", "..." : "..." },
  "route": { "..." : "..." }
}
```

**禁止在主语言文件中放入组件特有的扁平中文 key。**

### 第 5 步：确认 Generator 输出

1. 执行 `dotnet build YTStd.slnx` 触发 YTStdI18n.Generator
2. 确认 `locales/generated/{locale}.json` 已生成
3. 确认 JSON 格式为 `{ "18001": "用户名已存在", ... }`（ErrorCode→文本映射）

### 第 6 步：配置 DevExtreme 组件本地化

**必须先查阅 dxdocs**（见上方"DevExpress 文档查阅"章节）。

基于 dxdocs 文档配置 DevExtreme 内置文本的本地化：

```typescript
// 参考示例（以 dxdocs 文档为准）
import { locale, loadMessages } from 'devextreme/localization'
import zhMessages from 'devextreme/localization/messages/zh'
import jaMessages from 'devextreme/localization/messages/ja'

// 加载 DevExtreme 内置文本的翻译
loadMessages(zhMessages)
loadMessages(jaMessages)

// 设置当前 locale
locale('zh')
```

DevExtreme 的分页器（"第 X 页 / 共 Y 页"）、过滤器、排序提示等内置文本通过 DevExtreme 自身 locale 机制处理，不通过 vue-i18n 的 `t()`。

### 第 7 步：实现语言切换

1. 创建语言切换组件或在布局 header 中集成
2. 语言优先级（从高到低）：用户个人偏好 → 租户默认语言 → 系统默认（zh-CN）
3. 切换后立即生效要求：
   - 更新 `i18n.global.locale.value`
   - 更新 DevExtreme `locale()`
   - 保存用户偏好到后端
4. 页面无需刷新即可切换语言

### 第 8 步：在 `main.ts` 中注册 i18n

```typescript
import { i18n } from './locales'

const app = createApp(App)
app.use(i18n)
```

## 翻译归属规则（快速参考）

详见 `03-frontend/06-i18n-execution.md` 第四节。

```
某个 i18n key 放哪里？
  ↓
仅在单个组件中使用？
  ├── 是 → *.vue.{locale}.json（组件级）
  └── 否（多组件共用）
        ↓
      通用操作（确定/取消/保存/删除/查询）？
        ├── 是 → common/{locale}.json
        └── 否 → 枚举/状态 → enums/{locale}.json，其他 → common/{locale}.json
```

### null 值约定

```json
// XxxView.vue.zh-CN.json
{
  "平台用户管理": "平台用户管理",   // 组件自有 → 写具体值
  "查询": null,                      // 复用 → 写 null，由 common 提供
  "删除": null                       // 复用 → 写 null，由 common 提供
}
```

## notifySuccess / confirmAction 的 i18n 规则

```typescript
// ✅ 正确 — 仅传 i18n key，useNotify 内部调用 i18n.global.t()
notifySuccess('创建成功')
confirmAction('确认启用此用户')
confirmDelete(item.Name)

// ❌ 错误 — 禁止双重 t()
notifySuccess(t('创建成功'))
```

由于 `useNotify` 使用 `i18n.global.t()`（全局作用域），notify/confirm 使用的 key 必须在全局可见范围内。通用提示放 `common/`，模块特有提示放组件级文件（会被 `import.meta.glob` 自动加载到全局）。
