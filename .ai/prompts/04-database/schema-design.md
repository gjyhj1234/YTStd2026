# 表结构设计提示词

## 目标

设计或审查数据库表结构，确保符合命名规范、性能要求和业务需求。

---

## 适用范围

新建或修改数据库表结构时使用。

---

## 前置阅读

- `.ai/rules/database.md`
- `.ai/rules/naming.md`

---

## 输入

- 业务需求描述
- 已有的数据字典（如有）

---

## 输出

- 数据字典更新（`docs/{Module}/database_dictionary.md`）
- 对应的实体设计建议

---

## 执行步骤

1. 分析业务需求，确定需要的表
2. 按前缀体系确定表名
3. 设计字段（名称、类型、约束）
4. 设计索引（唯一索引、普通索引）
5. 设计表间关系
6. 审计字段按需添加
7. 输出数据字典格式文档
8. 验证命名规范一致性

---

## 前缀体系速查

| 前缀 | 适用范围 |
|------|---------|
| `plt_` | 平台管理 |
| `tnt_` | 租户相关 |
| `saas_` | SaaS 业务 |
| `sys_` | 系统配置 |
| `log_` | 日志审计 |
| `ntf_` | 通知 |
| `api_` | API 集成 |
| `stg_` | 存储 |

---

## 约束

- 所有规则参照 `.ai/rules/database.md`
- 表名必须有前缀
- 字段名必须 snake_case
- 禁止裸 `tenant_id`

---

## 验收标准

- [ ] 表名符合前缀体系
- [ ] 字段名符合 snake_case 规范
- [ ] 索引命名正确
- [ ] 审计字段完整
- [ ] 数据类型选择合理
- [ ] 无命名冲突
