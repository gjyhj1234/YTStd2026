# 前端生产级默认能力清单

> 当业务提示词没有把需求写全时，Agent 不能把缺失功能默认为“不需要”。本文件定义在提示词缺口下必须自动补齐的生产级默认能力。

---

## 一、适用原则

| 情况 | 处理规则 |
| ------ | --------- |
| 需求明确要求启用/禁用某能力 | 以显式需求为准 |
| 需求未提及，但属于生产级后台基础能力 | 按本文件默认补齐 |
| 默认能力与后端接口或现有架构冲突 | 停下说明冲突并请求确认 |
| 默认能力与业务明显不适配 | 在“需求补全矩阵”中明确标注为“不适用” |

---

## 二、所有页面的默认能力

所有前端页面默认必须显式决策以下能力：

| 能力 | 默认值 | 说明 |
| ------ | ------ | ------ |
| 页面标题 | 必须 | 页面主标题不可省略 |
| 功能说明入口 | 必须 | 使用功能说明卡片或等价共享资产 |
| 操作指引入口 | 必须 | 使用操作指引抽屉或等价共享资产 |
| 加载态 | 必须 | 首屏加载和提交中状态都要有 |
| 空态 | 必须 | 列表/详情无数据时有明确提示 |
| 错误态 | 必须 | 网络错误或业务失败不能静默 |
| 响应式 | 必须 | 至少覆盖桌面、平板、手机 |
| i18n | 必须 | 所有用户可见文本纳入 i18n |
| data-testid | 必须 | 关键元素必须可稳定定位 |

## 二点五、设计审查默认产物

所有前端业务提示词默认必须先产出“可视化设计审查包”，再进入实现描述。该审查包使用 Mermaid + Markdown 表格，保证 AI 可生成、人工可直观看、code review 可核对。

| 产物 | 默认规则 | 目标 |
| ------ | --------- | ---- |
| 页面结构蓝图 | 必须 | 看清页面区块、弹窗、抽屉、工具栏、列表、分页是否齐全 |
| 关键交互流 | 必须 | 看清查询、创建、编辑、危险操作、异常路径是否闭环 |
| 字段覆盖矩阵 | CRUD 页必须 | 看清字段是否覆盖查询、列表、详情、新增、编辑以及必填/只读/校验 |
| 操作/API 对照矩阵 | 只要存在操作按钮或工具栏动作就必须 | 看清每个操作是否有权限、API、确认、成功反馈、E2E 断言 |
| 状态反馈矩阵 | 必须 | 看清加载、空态、错误、403、提交中是否都有用户反馈 |

默认检查规则：

1. 审查包缺任一产物，提示词视为不完整。
2. 页面区块、字段、操作、状态必须能在正文章节与审查包之间双向对齐。
3. 如果 Mermaid 无法渲染，仍必须保留 Mermaid 源码，禁止退化为长段落纯文本。

## 三、CRUD 管理页默认能力

### 3.1 列表页默认能力

| 能力 | 默认规则 |
| ------ | --------- |
| 查询区 | 默认存在；如果查询字段超过 2 个，必须显式判断是否需要高级查询 |
| DataGrid | 默认使用 `DxDataGrid`，除非业务天然树形，否则不得自写表格 |
| 分页 | 默认启用，页大小至少声明 `[10, 20, 50, 100]` 是否适用 |
| 默认排序 | 必须显式声明；未声明视为缺项 |
| 焦点行 | 创建/编辑成功后默认需要定位到最近更新记录 |
| 列宽 | 每列必须声明固定宽度、最小宽度或 auto 规则 |
| 移动端列隐藏 | 默认启用 `column-hiding-enabled` 并声明隐藏优先级 |
| 固定列 | 操作列默认固定在右侧；主标识列默认评估是否固定 |
| 列显示/隐藏 | 必须显式决策是否启用列选择器或固定列方案 |
| 加载面板 | 默认启用 |
| 空数据文案 | 默认启用 |

### 3.2 表单默认能力

| 能力 | 默认规则 |
| ------ | --------- |
| 提交前验证 | 默认必须调用 `formInstance.validate()` |
| 提交 loading | 默认必须有 `submitting` 或等价状态 |
| 唯一性校验 | 所有唯一字段默认需要异步验证；编辑场景默认排除当前 Id |
| placeholder/helpText | 默认需要显式决策是否存在 |
| 成功行为 | 默认关闭弹窗、刷新列表、定位焦点行 |
| 失败行为 | 默认不关闭弹窗，保留用户输入 |
| 危险字段 | 默认需要明确禁用条件、显隐条件 |

### 3.3 密码与凭据相关默认能力

只要表单涉及密码、密钥、令牌、验证码等敏感输入，默认必须显式决策：

1. 是否需要二次输入确认。
2. 是否需要可见性切换。
3. 是否需要失败次数控制或验证码。
4. 是否需要危险操作确认。
5. 是否需要一次性展示结果并提醒保存。

如果业务提示词没写，**默认要求至少做出决策并记录理由**，不能直接忽略。

## 四、登录/认证页默认能力

登录、注册、重置密码、修改密码等认证类页面，默认必须显式决策以下能力：

| 能力 | 默认规则 |
| ------ | --------- |
| 回车提交 | 默认启用 |
| 登录中禁用 | 默认启用 |
| 错误凭据提示 | 默认启用且不清空表单 |
| 连续失败保护 | 默认三次失败后出现验证码或等价校验 |
| RequirePasswordReset 分支 | 默认必须处理 |
| 语言切换 | 默认必须存在 |
| Redirect 回跳 | 默认必须处理 |
| 安全提示 | 默认明确告知锁定、禁用、过期等状态 |

## 五、权限与入口默认能力

这是历史上最容易“看起来生效、实际不可用”的部分，必须按默认规则补齐。

### 5.1 默认规则

1. **菜单入口可见性**、**路由可达性**、**按钮显隐** 是三个独立层次，必须分别声明。
2. 如果用户拥有某页面的“查看/列表/查询”能力，默认应能看到对应入口，除非业务明确要求入口隐藏但支持直达。
3. 无权限时默认展示 403 或无权限提示，不能白屏或直接把应用主入口隐藏掉。
4. 子功能权限默认不能反向导致父菜单整个消失，除非父菜单下所有子项都不可访问。

### 5.2 默认必须输出权限入口可见性矩阵

| 层级 | 权限码 | 默认行为 |
| ------ | -------- | --------- |
| 菜单入口 | `{module}.list` 或等价权限 | 决定菜单是否显示 |
| 页面进入 | `{module}.list` 或 `{module}.detail` | 决定 URL 可否进入 |
| 查询能力 | `{module}.list` | 决定搜索与列表是否可见 |
| 行操作 | `{module}.update/delete/...` | 决定按钮显隐与可点击 |

## 六、测试默认能力

所有页面默认需要以下测试维度，业务提示词必须逐项说明是否覆盖：

1. 页面渲染。
2. 关键路径操作。
3. 表单验证。
4. 权限入口与按钮显隐。
5. 响应式布局。
6. 多语言切换。
7. 错误路径或空态。
8. 关键字段标签与绑定对应关系。

业务提示词或任务切片中，必须至少填写如下矩阵：

| 类别 | 显式需求 | 生产默认补全 | 待确认项 | 明确不做项 |

| DataGrid 能力 | | | | |
| 表单校验 | | | | |
