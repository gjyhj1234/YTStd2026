# 前端任务切片规范（适配 Copilot Agent 1 小时限制）

> 本文件定义前端任务的最小执行单元。重点不是“一个模块一个任务”，而是“一个模块拆成多个可验证 slice”。

---

## 一、核心原则

| 原则 | 要求 |
| ------ | ------ |
| Slice 优先 | 模块提示词只是 Epic，真正执行的是 slice |
| 时间受限优先 | 单个 slice 必须可在 1 小时限制内完成 |
| 验证闭环优先 | 每个 slice 都要有独立的构建/测试/快照产物 |
| 共享文件降冲突 | 一个 slice 尽量只触达一个模块目录和极少量共享文件 |
| 任务过大立即再拆 | 预计超出时长就继续拆，不能硬做 |

---

## 二、切片层级

| 层级 | 含义 | 例子 |
| ------ | ------ | ------ |
| Epic | 一个完整模块或完整页面的总任务 | `F2-2 平台用户管理` |
| Slice | 一次 Agent 会话的实际执行单元 | `F2-2C 表单与校验` |
| Patch | 审查或测试失败后的定点修复 | `F2-2C-fix1` |
| Verify | 仅做验证与文档同步的末端切片 | `F2-2E E2E 与自审` |

---

## 三、标准切片模型

### 3.1 CRUD 管理页标准切片

大多数后台管理页必须拆成以下切片：

| 切片 | 目标 | 典型文件数 | 典型耗时 |
| ------ | ------ | ----------- | --------- |
| A. 合同与骨架 | 需求补全矩阵、资产复用决策、路由入口、页面骨架、data-testid 规划 | 2-5 | 20-30 分钟 |
| B. 搜索与列表 | 查询区、DataGrid、分页、基础数据加载、空态/加载态 | 3-6 | 25-35 分钟 |
| C. 表单与校验 | 新增/编辑弹窗、字段校验、唯一性校验、提交 loading | 3-6 | 25-35 分钟 |
| D. 状态动作与权限 | 详情、状态流转、按钮显隐、菜单入口、路由守卫、危险操作确认 | 2-5 | 20-35 分钟 |
| E. E2E 与收尾 | Playwright、self-review、README 同步、任务快照更新 | 2-5 | 20-35 分钟 |

> 如果模块还有复杂关系管理（例如成员分配、权限树、生命周期事件），必须再拆出 `D1`、`D2` 子切片，禁止挤进同一个 D 切片。

### 3.2 登录/认证页标准切片

| 切片 | 目标 |
| ------ | ------ |
| A. 页面骨架与语言切换 | 基础布局、标题、副标题、语言切换 |
| B. 表单与认证流程 | 用户名/密码、提交、错误处理、跳转 |
| C. 安全补全 | 失败计数、验证码、密码重置分支、data-testid |
| D. E2E 与收尾 | 无登录测试、响应式、多语言、验证证据 |

### 3.3 纯展示/只读页标准切片

| 切片 | 目标 |
| ------ | ------ |
| A. 页面骨架与数据加载 | 页面框架、请求、空态、加载态 |
| B. 可视化与交互 | 图表、卡片、筛选器、跳转 |
| C. E2E 与收尾 | 测试、自审、快照 |

---

## 四、切片尺寸上限

### 4.1 默认上限

| 维度 | 上限 |
| ------ | ------ |
| 新增/修改 `.vue` 文件 | ≤ 2 |
| 新增/修改 `.ts` 文件 | ≤ 4 |
| 新增/修改共享文件 | ≤ 2 |
| DevExtreme 新组件种类 | ≤ 3 |
| 新增验证场景 | ≤ 8 |
| 新增 E2E 用例 | ≤ 10 |

### 4.2 触发再拆分的条件

出现以下任一情况，必须再拆：

1. 同时改路由、菜单、列表、表单、权限、E2E。
2. 同时新增 2 个以上弹窗/抽屉。
3. 同时引入 4 种以上 DevExtreme 复杂组件。
4. 需要修改多个共享组件或多个全局配置文件。
5. 预计需要完整 E2E 迭代超过 2 轮。
6. 需要等待后端能力、权限数据或复杂联动数据准备。

## 五、切片命名规范

### 5.1 推荐命名

| 类型 | 示例 |
| ------ | ------ |
| 模块 Epic | `F2-2 平台用户管理` |
| Slice | `F2-2B 平台用户管理 - 搜索与列表` |
| Patch | `F2-2B-fix1 列表选择器修复` |
| Verify | `F2-2E 平台用户管理 - E2E 与自审` |

### 5.2 GitHub Issue 标题规范

```text
[FE][F2-2B] 平台用户管理 - 搜索与列表
[FE][F2-2E] 平台用户管理 - E2E 与自审
```

## 六、模块级业务提示词与切片关系

业务提示词必须承担 **Epic 合同** 的职责，不承担“一次性全做完”的职责。

| 文件 | 职责 |
| ------ | ------ |
| `08-platform/frontend/00xx_*.md` | 说明完整模块的目标、字段、规则、验收 |
| `.ai/tasks/platform-frontend/*.md` | 说明当前正在做哪个 slice、做到哪里、还有什么待办 |
| `.ai/workspace/session-summary-*.md` | 记录这轮真的做了什么、测了什么、下一轮从哪接 |

---

## 七、每个切片必须包含的输出

1. 当前切片编号和状态。
2. 本切片修改文件清单。
3. 本切片完成了哪些功能点。
4. 本切片未完成哪些功能点。
5. 实际运行的验证命令。
6. 如果需要下一切片，明确写出依赖和进入条件。

---

## 八、切片进入条件与退出条件

### 8.1 进入条件

切片开始前，必须满足：

1. 依赖切片已完成。
2. 需求补全矩阵已填写。
3. 组件复用决策表已填写。
4. dxdocs 查询问题已列出。
5. 共享文件冲突已确认。

### 8.2 退出条件

切片结束前，必须满足：

1. 当前切片声明的功能点已完成。
2. 构建通过或明确说明未通过原因。
3. 已更新 `.ai/tasks/...` 当前任务文件。
4. 已输出 `.ai/workspace/session-summary-*.md`。
5. 若发现新通用问题，已更新对应提示词文件。

## 九、示例：F2-2 平台用户管理切片拆分

| 切片 | 内容 | 进入条件 | 退出条件 |
| ------ | ------ | --------- | --------- |
| F2-2A | 需求合同、页面骨架、data-testid 规划、入口权限矩阵 | 布局层完成 | 页面可进入，标题/骨架存在 |
| F2-2B | 查询区、DataGrid、分页、远程加载 | A 完成 | 列表加载和搜索工作 |
| F2-2C | 新增/编辑表单、字段校验、唯一性校验 | B 完成 | 创建/编辑成功路径可走通 |
| F2-2D | 查看、启用/禁用、重置密码、批量操作、按钮权限 | C 完成 | 所有动作路径和显隐逻辑正确 |
| F2-2E | E2E、自审、README/快照同步 | D 完成 | 测试与自审留痕完成 |
