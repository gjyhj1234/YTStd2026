# 前端任务拆分规范

> 本文件定义前端任务的标准拆分粒度、依赖关系、执行顺序和验收方式。
> 所有前端任务必须按本规范拆分后再执行，禁止"一次性实现全部"。

---

## 一、拆分原则

### 1.1 单次任务粒度

每次任务的最大粒度为 **一个可独立验收的功能单元**。具体约束：

| 维度 | 上限 |
|------|------|
| 新增/修改 Vue 文件数 | ≤ 3 个 |
| 新增/修改 TS 文件数 | ≤ 5 个 |
| 新增语言文件数 | 不限（但必须完整，5 种语言齐备） |
| 涉及 DevExtreme 组件种类 | ≤ 5 种 |

### 1.2 不可跨越的任务边界

以下内容不得合并到同一任务中：

- 脚手架搭建 与 业务页面实现
- 布局组件 与 业务页面实现
- 不同业务模块的页面实现（如"用户管理"与"角色管理"不得合并）
- i18n 基础设施 与 业务页面 i18n
- axios 封装 与 业务 API 调用

### 1.3 必须合并的内容

以下内容必须在同一任务中完成：

- 一个页面的 `.vue` 文件 + 对应 5 个语言文件
- 一个页面的 API 封装（`api/{module}.ts`）+ 类型定义（`types/{module}.ts`）
- 一个页面的静态配置文件（`columns.ts`、`query-form.ts` 等）+ 对应语言归属

## 二、标准任务层级

前端开发按以下层级拆分任务，**必须按层级顺序执行**：

| F0-3 | i18n 基础设施 | F0-1 | `src/locales/` 结构 + 自动加载机制 |

## 三、任务拆分 checklist

为任何新的前端业务模块创建提示词时，必须按以下 checklist 确认拆分完整性：

- [ ] 明确了页面结构（主标题、副标题、功能说明区、查询区、表格区、分页区等）
- [ ] 明确了所有查询条件（通用 + 高级）
- [ ] 明确了所有表格列（数据字段、格式化、状态颜色）
- [ ] 明确了所有操作按钮（名称、图标、区域、权限、条件、行为）
- [ ] 明确了所有弹窗/抽屉（标题、表单字段、验证规则）
- [ ] 明确了所有表单字段（名称、类型、必填、长度、格式、唯一性、默认值）
- [ ] 明确了所有 API 端点（HTTP 方法 + URL + 请求/响应体）
- [ ] 明确了所有静态配置文件（columns.ts、query-form.ts、form.ts 等）
- [ ] 明确了所有国际化 key（包含静态配置文件中的文本）
- [ ] 明确了验收 checklist（逐项功能点，非笼统描述）

---

## 四、依赖关系强制规则

1. **层级 0 必须全部完成后才能开始层级 1**
2. **层级 1 必须全部完成后才能开始层级 2**
3. 层级 2 内部的任务**可以并行**（见 `02-parallel-execution.md`）
4. 层级 3 必须在层级 2 全部完成后执行

---

## 五、单任务输出标准

每个任务完成后必须输出：

1. **新增文件清单**（含路径）
2. **修改文件清单**（含路径和变更说明）
3. **验收 checklist 执行结果**（逐项标记通过/失败）
4. **Code Review 自检结果**（见 `00-governance.md` 第四节）
5. **乱码检查结果**（见 `00-governance.md` 第五节）
6. **编译结果**（`npm run build` 输出摘要）
