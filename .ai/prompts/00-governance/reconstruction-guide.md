# 重建指南

## 目标

指导如何使用本提示词体系，通过 GitHub Agents 从零开始重建整个解决方案。

---

## 适用范围

项目初始化或完全重建场景。

---

## 前置条件

1. 仓库已包含完整的 `.ai/` 提示词体系
2. 仓库已包含基础框架模块（YTStdSqlBuilder、YTStdLogger、YTStdAdo、YTStdEntity、YTStdI18n）
3. `dotnet build YTStd.slnx` 通过
4. GitHub Agents 已配置可用

---

## 重建步骤

### 第 1 步：确认规则（人类）

1. 审查 `.ai/rules/` 下所有规则文件
2. 审查 `.ai/context/` 下所有上下文文件
3. 审查 `.ai/system/` 下所有系统文件
4. 如需调整，直接修改对应文件
5. 确认数据字典 `docs/TenantPlatform/database_dictionary.md` 正确

### 第 2 步：启动第一轮 Agent

向 Agent 发送以下指令：

```markdown
请阅读以下文件：
- `.ai/system/agent-contract.md`
- `.ai/rules/global.md`
- `.ai/context/tech-stack.md`
- `.ai/prompts/00-governance/phase-plan.md`

然后执行 Phase P0（协作规则确认）和 Phase P1（数据库设计验证）。
```

### 第 3 步：按阶段推进

每轮 Agent 执行一个或多个阶段（取决于复杂度），按 `phase-plan.md` 中的顺序推进。

### 第 4 步：人类审查

在每个标记的"人类介入点"暂停，人类审查后继续。

### 第 5 步：持续推进直到完成

重复第 3-4 步，直到所有阶段完成。

---

## 重建优先级

```
第 1 优先级：平台底座
  ├── 数据库设计 (P1)
  ├── 实体建模 (P2)
  ├── 初始化数据 (P3)
  └── 后端基础设施 (P4)

第 2 优先级：核心 API
  ├── 认证 API
  ├── 平台用户/角色/权限 API
  └── 租户管理 API

第 3 优先级：扩展 API
  ├── 套餐/订阅/计费 API
  ├── API 集成 API
  └── 运营/审计/通知 API

第 4 优先级：前端
  ├── 前端骨架
  ├── 核心页面
  └── 扩展页面

第 5 优先级：完善
  ├── 国际化
  ├── 测试补齐
  ├── Postman 集合
  └── 文档整理
```

---

## 中断恢复

如果重建过程中断：

1. 查看最新的 `.ai/workspace/session-summary-*.md`
2. 确认代码库当前状态
3. 从上次中断的阶段继续
4. 参考 `.ai/system/session-handoff.md`

---

## 人类必须确认的节点

| 节点 | 原因 |
|------|------|
| 数据字典变更 | 影响全部实体和 API |
| 新增外部依赖 | 可能影响 AOT 兼容性 |
| 中间件管道顺序 | 影响所有请求处理 |
| 权限模型设计 | 安全关键决策 |
| 数据库表结构变更 | 不可逆操作 |

## Agent 可连续执行的操作

| 操作 | 原因 |
|------|------|
| 创建 DTO | 低风险，可编译验证 |
| 创建应用服务 | 低风险，可测试验证 |
| 创建 API 端点 | 低风险，可编译验证 |
| 创建前端页面 | 低风险，可编译验证 |
| 创建测试 | 低风险，可运行验证 |
| 添加注释 | 无风险 |
| 更新文档 | 低风险 |
