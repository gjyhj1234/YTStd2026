# API 文档提示词

## 目标

为已完成的 API 端点生成完整的接口文档。

---

## 适用范围

生成或更新 API 文档时使用。

---

## 前置阅读

- `.ai/rules/documentation.md`
- `.ai/rules/api-design.md`

---

## 输入

- 已完成的 API 端点代码
- 已完成的 DTO 定义

---

## 输出

- `docs/{Module}/API.md`

---

## 执行步骤

1. 扫描所有 Endpoint 文件
2. 提取路由、HTTP 方法、权限码
3. 提取请求和响应 DTO 字段
4. 按模块分组输出文档
5. 验证文档与代码一致

---

## 文档格式

每个端点按以下格式记录：

```markdown
### {HTTP方法} {路径}

{端点说明}

**权限码**：`{权限码}`

**请求参数/体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|

**响应**：

| 字段 | 类型 | 说明 |
|------|------|------|

**错误码**：

| 错误码 | 说明 |
|-------|------|

**示例**：

请求：
```json
{ ... }
```

响应：
```json
{ ... }
```
```

---

## 约束

- 文档与代码严格一致
- 所有端点有权限码说明
- 所有请求字段有类型和必填标注
- 所有错误码有说明

---

## 验收标准

- [ ] 文档覆盖所有 API 端点
- [ ] 路径与代码一致
- [ ] 字段与 DTO 一致
- [ ] 无乱码
