# 架构文档提示词

## 目标

生成或更新模块的架构文档，包含功能架构、技术架构和部署架构。

---

## 适用范围

生成或更新架构文档时使用。

---

## 前置阅读

- `.ai/rules/documentation.md`
- `.ai/context/tech-stack.md`
- `.ai/context/project-structure.md`

---

## 输入

- 已完成的代码
- 业务需求

---

## 输出

- `docs/{Module}/architecture.md`

---

## 文档包含内容

1. 概述
2. 功能/菜单架构图
3. 技术架构说明
4. 分层结构说明
5. 模块依赖关系
6. 数据库表清单
7. API 端点清单
8. 中间件管道说明
9. 缓存策略说明
10. 安全策略说明
11. 部署说明

---

## 约束

- 架构文档反映实际实现
- 使用 Markdown 格式
- 使用 ASCII 图表（非外部工具依赖）

---

## 验收标准

- [ ] 架构文档与实际代码一致
- [ ] 结构完整
- [ ] 无乱码
