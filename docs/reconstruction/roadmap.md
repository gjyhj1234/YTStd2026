# 重建路线建议

## 概述

本文档提供从零使用 GitHub Agents 重建整个解决方案的推荐路线，分为 6 个主要阶段、13 个子阶段。

---

## 阶段 1：基础建立（预计 2-3 轮 Agent）

### 1.1 协作规则确认

- Agent 阅读所有 `.ai/system/` 和 `.ai/rules/` 文件
- 确认技术栈和约束
- 验证 `dotnet build YTStd.slnx` 通过
- **人类确认**：规则无需调整

### 1.2 数据库设计验证

- 审查数据字典（`docs/TenantPlatform/database_dictionary.md`）
- 按 `.ai/rules/database.md` 验证命名规范
- 补全缺失的表和字段
- **人类确认**：数据字典变更

### 1.3 实体建模（分批）

- 批次 1：平台管理实体（plt_*）
- 批次 2：租户实体（tnt_*）
- 批次 3：资源与配置实体（sys_*）
- 批次 4：SaaS 业务实体（saas_*）
- 批次 5：运营支撑实体（log_*、ntf_*、api_*、stg_*）
- 每批编译验证 Source Generator

| 阶段 2 | 3-5 | 中间件管道确认 |

| **合计** | **22-35** | — |

---

## 使用此路线的方式

1. 人类按阶段顺序向 Agent 发送任务
2. 每个任务引用对应的规则文件和提示词文件
3. Agent 执行后输出会话总结
4. 人类在标记的介入点审查确认
5. 确认后继续下一个任务
6. 如果某个任务失败，Agent 回滚并报告，人类决定下一步
